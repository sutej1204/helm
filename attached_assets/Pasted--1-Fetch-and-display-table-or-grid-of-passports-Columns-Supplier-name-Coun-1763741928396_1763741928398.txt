6. Detail View (5-Layer Passport)
==================================================

Create `SupplierPassportDetail` for `/supplier-passports/:id`.

Structure:

- Top header/card:
  - Supplier name, country, industry
  - Status badge (Active/Pending/On Watchlist)
  - Risk band badge
  - KYC status badge
  - SCF eligible indicator

- Below header, use either:
  - 5 collapsible panels, OR
  - 5 tabs
  representing each layer:

(1) Identity & Compliance
- Show all compliance fields.
- Visually emphasize high-risk values (e.g., PEP flag, high ESG/forcedLaborRisk, sanctionsHits > 0).
- Display expiry date and “KYC refresh needed” if in the past.

(2) Contract Performance & Value Leakage
- Show contracted payment terms, discounts, rebates.
- Show average price deviation %, rebate utilisation %, identified leakage annual $.
- Show “Why you’re leaking” as a short explanatory line composed from the data (e.g., “Paying on average 12 days earlier than terms; low rebate utilisation”).

(3) Payment Behavior & Working Capital
- Show:
  - Actual average payment days vs contracted days.
  - Early/late/on-time percentages.
  - Payment volatility score.
  - Working capital opportunity score.
- Optionally show a simple visual bar or comparison (using existing UI patterns).

(4) Financing & Capital Access
- Show:
  - SCF, invoice finance, dynamic discounting eligibility.
  - Risk band, indicative funding limit, spread.
  - Funding readiness score.
  - Blocking barriers list (“KYC expired”, “insufficient history”, etc.).
- Show a small label such as:
  - “Ready for financing”
  - or “Requires compliance update before funding”

(5) Automation & Agentic Actions
- Show:
  - Supplier credits earned / pending
  - Rebates due
  - Cash flow recovery to date
  - Margin leakage alert %
  - List of `agenticActionFlags` as badges
- Show an “Action history” log:
  - Simple array with date + description (hardcode a few examples in mock data).
- Add small CTAs:
  - Buttons like “Trigger rebate recovery”, “Notify procurement”, “Offer financing”, “Initiate KYC refresh”.
  - For now these can:
    - Open a simple modal confirming the action, and
    - Append a new entry to the local action history array for that supplier in the mock data.

Editing:
- Provide an **Edit** button on the page.
- Edit mode toggles fields into form controls grouped by layer.
- On save, call `updateSupplierPassport`.
- Handle basic validation for numbers (percentages 0–100, no negatives, etc.).

7. Create / Edit Flow
==================================================

Create `SupplierPassportCreate` for `/supplier-passports/new` (or use a modal if that’s normal in this project).

Form behaviour:
- Capture:
  - Basic identity: name, country, industry, initial status.
  - Minimal required compliance fields (kycStatus, ESG risk, forcedLaborRisk, etc.).
  - Key contract fields (contracted terms, initial leakage estimate).
  - Key payment behaviour fields (you can default some values).
  - Financing basics: scfEligible, riskBand, indicativeFundingLimitUSD.
  - Automation basics: supplierCreditsEarnedUSD (default 0), etc.
- Use sensible defaults where appropriate to keep the form manageable.
- On submit:
  - Call `createSupplierPassport`.
  - Redirect to the new detail view.

Consider reusing components between create and edit forms.
